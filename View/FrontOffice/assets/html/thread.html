Rebuild a premium hero section from a reference and explain every design decision.

This page combines the task brief and its discussion space, so users can understand the work, track requirements, and talk inside the same context.

ADN impact: High
Difficulty Intermediate Best for learners already building real interfaces.
Expected time 3 hours Enough time for a solid first version and rationale note.
Deadline Today / 6:00 PM Manager review starts right after submission closes.
Task brief

What the task asks for

Build a premium front-end hero section inspired by a reference, but do not copy it mechanically. Your submission should show hierarchy, motion restraint, spacing logic, and a short written explanation of why each design decision supports the experience.

Main outcome A finished hero implementation with responsive behavior and polished visual hierarchy.
Required note A short reasoning summary explaining typography, spacing, contrast, and call-to-action choices.
One more step Include the improvement you would still make if you had more time.
Deliverables
Responsive hero layout

Desktop and mobile versions should both feel intentional, not collapsed.

Design rationale

Explain why the structure feels premium and what visual problems you solved.

Improvement note

Name the next refinement you would push if you had a second iteration.
