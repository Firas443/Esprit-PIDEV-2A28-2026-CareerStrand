# CareerStrand – ADN Career Progression Platform

## Overview
This project was developed as part of the **PIDEV – 2nd Year Engineering Program** at **Esprit School of Engineering** during the academic year **2025–2026**.

**CareerStrand** is a web platform designed to help **students, beginners, and young professionals** prepare for the digital job market through a **structured progression journey**.

The platform is built around the concept of **ADN**, which represents the user’s **professional identity** inside the system.

Unlike an AI-generated profile, the **ADN** is built progressively through the user’s real journey on the platform:
- profile completion
- skills and preferences
- course enrollment and progress
- challenge submissions
- event participation
- opportunity applications

CareerStrand is not only a learning platform, a networking platform, or a job board.  
It is a **career progression ecosystem** that helps users move from **preparation** to **real opportunities**.

---

## Main Objective
CareerStrand aims to reduce the gap between **learning** and **professional opportunities** by offering one integrated platform where users can:

- build a professional profile
- learn through courses
- prove skills through practical challenges
- engage in workshops, hackathons, and events
- apply to internships, junior roles, and beginner opportunities

---

## Core Concept: ADN
In CareerStrand, **ADN** is the digital reflection of the user’s professional journey.

It is shaped by:
- personal profile
- skills and preferences
- education history
- challenge performance
- event participation
- opportunity applications

The ADN is therefore a **dynamic identity built through progression**, not a static profile and not an automatically generated result.

## Platform Flow
The platform follows a structured user journey:

**Register → Complete Profile → Enroll in Courses → Submit Challenges → Participate in Events → Apply to Opportunities**

This progression is one of the strongest aspects of the project because it reflects user growth and readiness.

## Main Actors

### User
The main actor of the platform.  
This can be:
- a student
- a beginner
- a junior freelancer
- a young professional

The user learns, submits work, joins events, and applies to opportunities.

### Manager
The manager is responsible for publishing and managing opportunities and events.  
This can represent:
- a recruiter
- a startup founder
- an opportunity provider

### Admin
The admin supervises the platform and manages:
- users
- profiles
- courses
- challenges
- events
- opportunities
- platform content

## UML Logic
The system is based on the following main entities:

- **User**
- **Profile**
- **Course**
- **Enrollment**
- **Challenge**
- **Submission**
- **Event**
- **Participation**
- **Opportunity**
- **Application**

### Main Relationships
- One **User** has one **Profile**
- One **User** can have many **Enrollments**
- One **User** can have many **Submissions**
- One **User** can have many **Participations**
- One **User** can have many **Applications**

This structure models the user’s progression through the platform.
